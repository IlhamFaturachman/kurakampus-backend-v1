Saya telah memeriksa seluruh codebase Anda dan menemukan bahwa **saat ini tidak ada endpoint yang menggunakan prefix** **`/api`**. Konfigurasi global untuk prefix ini belum diterapkan.

Berikut adalah detail temuan dan rencana perbaikan untuk menstandarisasi seluruh endpoint dengan awalan `/api`.

### 🔍 Hasil Audit Routing

| Controller                  | Path Saat Ini (Salah) | Path Seharusnya (Benar) | Status           |
| :-------------------------- | :-------------------- | :---------------------- | :--------------- |
| **AuthController**          | `/auth/*`             | `/api/auth/*`           | ❌ Missing Prefix |
| **OrganizationsController** | `/organizations/*`    | `/api/organizations/*`  | ❌ Missing Prefix |
| **AppController**           | `/`                   | `/api`                  | ❌ Missing Prefix |

**Lokasi Masalah:**

* **`src/main.ts`**: Tidak ditemukan konfigurasi `app.setGlobalPrefix('api')`.

* **`src/auth/auth.controller.ts`**: Menggunakan `@Controller('auth')` (Sudah benar, hanya butuh global prefix).

* **`src/organizations/organizations.controller.ts`**: Menggunakan `@Controller('organizations')` (Sudah benar, hanya butuh global prefix).

***

### 🛠 Rencana Implementasi

Saya akan melakukan perubahan pada **satu file kunci** untuk memberlakukan prefix `/api` secara global ke seluruh aplikasi secara otomatis. Anda tidak perlu mengubah satu per satu file controller.

#### 1. Modifikasi `src/main.ts`

Saya akan menambahkan `app.setGlobalPrefix('api')` tepat setelah inisialisasi aplikasi.

```typescript
// src/main.ts

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // [BARU] Set Global Prefix '/api'
  // Ini akan otomatis mengubah:
  // /auth/login -> /api/auth/login
  // /organizations -> /api/organizations
  app.setGlobalPrefix('api'); 

  // ... konfigurasi lainnya (Helmet, CORS, dll)
}
```

#### 2. Update Swagger Documentation

Swagger UI saat ini diakses di `/api/docs`. Dengan perubahan prefix global, saya akan memastikan konfigurasi Swagger tetap dapat diakses dan menampilkan endpoint yang sudah diperbarui (misalnya endpoint di dokumentasi akan otomatis tertulis `/api/auth/login`).
